# 美妆日报飞书定时推送设计说明

## 目标

每天北京时间 11:00，机器人检查前一自然日是否存在已发布、且群消息已经 Grace 审核通过的日报。符合条件时，机器人向一个固定飞书群推送摘要和网页详情入口；不符合条件时保持安静，不发送缺报、延迟或提醒消息。

本功能只负责分发已经确认的内容，不参与日报搜索、写作、审稿或网站发布决策。

## 已确认规则

- 推送对象为一个固定飞书群。
- 使用日报网页作为详情入口，不再发送 HTML 或 PDF 文件附件。
- 摘要条数不固定，根据当日信息价值决定。
- Grace 通知正式发布日报时，同步生成群消息预览。
- 只有 Grace 审核通过的群消息可以进入定时推送队列。
- 前一自然日没有符合条件的日报时，机器人不推送任何消息。
- 未确认稿、候选池、仅生成但未正式发布的 HTML 均不得推送。

## 方案选择

采用“GitHub Actions 定时任务 + 飞书群自定义机器人”。

当前网站已经通过 GitHub 仓库维护已发布日报清单。定时任务与该清单放在同一仓库，可以直接复用日报日期、发布状态和固定网页路由，无需新增常驻服务器。

飞书侧使用固定群的自定义机器人 Webhook。机器人只向所在群发送消息，不需要读取聊天记录、通讯录或其他飞书资源。Webhook 地址和签名密钥保存在 GitHub Secrets，不写入代码、发布清单或构建产物。

## 系统组成

### 发布清单

继续以 `site/data/published.json` 作为唯一发布门禁。日报记录新增可选的 `push` 字段：

```json
{
  "kind": "daily",
  "date": "2026-07-15",
  "title": "美妆竞对情报日报｜2026年7月15日",
  "source": "archive/日报/2026-07-15/beauty-daily-20260715.html",
  "push": {
    "status": "approved",
    "heading": "昨日重点",
    "items": [
      "第一条审核后的摘要。",
      "第二条审核后的摘要。"
    ],
    "detail_label": "查看完整日报"
  }
}
```

`push.status` 只允许以下状态：

- `draft`：已生成预览，等待 Grace 审核；定时任务不得发送。
- `approved`：Grace 已确认；允许在规定日期发送。
- `disabled`：该日报明确不推送。

日报记录没有 `push` 字段时，视为未审核，不发送。

### 群消息预览生成器

Grace 通知正式发布某日日报后，发布流程根据该日报的最终内容生成群消息预览。预览包含：

- 日报标题和日期；
- `昨日重点`；
- 数量可变的摘要列表；
- 对应日报网页入口。

摘要属于日报发布产物的一部分。Grace 可以增删、改写和调整顺序。审核通过后，最终文字原样写入发布清单，不在发送当天再次调用 AI 改写。

### 定时发送任务

GitHub Actions 使用 UTC 时间 `03:00` 触发，对应北京时间 `11:00`。任务按北京时间计算前一自然日，并执行以下判断：

1. 在发布清单中查找 `kind=daily` 且日期等于昨天的唯一记录。
2. 检查日报源文件存在，且 `push.status=approved`。
3. 检查摘要列表至少包含一条有效文字。
4. 根据配置的网站根地址生成 `/daily/YYYY-MM-DD/` 详情链接。
5. 组装飞书消息并发送到固定群机器人。

任一检查不通过时，任务正常结束且不调用飞书 Webhook。

### 飞书消息

第一版使用飞书富文本或交互卡片，正文结构为：

```text
【美妆竞对情报日报｜7月15日】

昨日重点：
1. ……
2. ……
3. ……

详细日报请查看网页端：
查看完整日报
```

摘要数量由 `push.items` 决定。详情按钮或链接只指向正式网站地址，不使用本机地址、临时 HTML 地址或附件地址。

## 审核与发送时序

1. Grace 通知“正式发布 YYYY-MM-DD”。
2. 完成日报网站发布，并生成群消息预览。
3. Grace 审核群消息文字。
4. 审核通过后将 `push.status` 改为 `approved`，并保存最终摘要。
5. 次日 11:00，定时任务发送该日报。

如果群消息在次日 11:00 之后才审核通过，系统不自动补发，避免过时消息突然进入群聊。需要补发时单独执行指定日期的人工发送流程，并在发送前再次确认。

## 安全与防误发

- GitHub Secrets 保存 `FEISHU_WEBHOOK_URL` 和可选的 `FEISHU_WEBHOOK_SECRET`。
- 日志不得输出完整 Webhook、签名密钥或消息鉴权信息。
- 定时任务只读取发布清单，不扫描草稿目录寻找“最新文件”。
- 消息中的日期来自目标日报记录，不使用运行时猜测标题日期。
- 同一天的定时任务设置并发锁，避免两个运行实例同时发送。
- 人工触发默认只生成预览；真实补发必须明确指定日期和发送开关。

第一版不引入数据库。正常定时任务每天只有一个触发实例；防并发锁用于避免同一时刻重复运行。若后续需要严格记录已发送状态、自动重试或向多个群分发，再增加独立发送台账。

## 异常处理

- 昨日无日报：安静退出。
- 日报已发布但群消息仍为 `draft`：安静退出。
- 摘要为空、网页地址配置缺失或记录重复：停止发送，并在任务日志中标记配置错误。
- 飞书返回明确失败：任务失败并保留错误摘要，但日志隐藏密钥。
- 飞书响应结果不确定：不进行无限自动重试，避免同一消息重复出现；转为人工检查。
- 网站暂时无法访问：不改写为附件发送，任务失败并等待人工处理。

## 验证

实现时至少覆盖以下测试：

1. 昨日存在 `approved` 日报时，生成数量可变、顺序正确的消息。
2. 昨日无日报时，不产生网络请求。
3. `draft`、`disabled` 或缺少 `push` 字段时，不产生网络请求。
4. 摘要为空、日期重复、网页根地址缺失时，任务明确失败。
5. 详情链接准确指向目标日期的正式日报页。
6. 飞书签名和请求体格式通过本地模拟验证。
7. 使用测试群进行一次人工触发测试，Grace 确认消息样式后再启用每日定时任务。

## 第一阶段交付范围

1. 发布清单的群消息字段和校验；
2. 群消息预览生成；
3. 飞书卡片或富文本组装；
4. GitHub Actions 每日 11:00 定时任务；
5. 安静跳过、并发保护和安全日志；
6. 测试群人工验证与正式群切换说明。

不包含自动 AI 摘要、多个群分发、私聊推送、已读统计、聊天指令查询日报或自动周报推送。
